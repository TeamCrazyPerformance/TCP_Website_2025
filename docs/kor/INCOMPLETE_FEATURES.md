### 1. [서버][보안] SSL 인증서 누락

**영향**: HTTPS가 구성되었으나 작동하지 않음  
**위치**: `reverse-proxy/certs/`  
**작업량**: 낮음

**이슈**: `.gitkeep` 파일만 존재하지만 nginx가 다음을 요구:
- `/etc/nginx/certs/fullchain.pem`
- `/etc/nginx/certs/privkey.pem`

**수정**:
```bash
# 방법 1: Let's Encrypt
certbot certonly --standalone -d your-domain.com

# 방법 2: 수동 인증서
cp /path/to/fullchain.pem reverse-proxy/certs/
cp /path/to/privkey.pem reverse-proxy/certs/
```

### 6.5. [프론트엔드][연동] 마이페이지가 백엔드 API에 연결되지 않음

**영향**: 사용자 프로필 기능이 작동하지 않음  
**위치**: 모든 `pages/mypage/*.jsx`  
**작업량**: 중간

**이슈**: 백엔드 엔드포인트가 존재하지만 프론트엔드에서 호출하지 않음

**영향받는 파일**:
- `Profile.jsx` - 하드코딩된 목업 프로필 데이터 사용, 저장 시 `alert()`
- `MyPageAccountSettings.jsx` - 목업 데이터, 실제 API 호출 없음
- `MyStudies.jsx` - 하드코딩된 스터디 데이터 배열
- `MyTeams.jsx` - 5개의 목업 팀이 포함된 하드코딩 배열

**사용 가능한 백엔드 API** (미사용 중):
```typescript
GET  /api/v1/mypage/profile      // 사용자 프로필 조회
PATCH /api/v1/mypage/profile     // 프로필 수정

GET  /api/v1/mypage/account      // 계정 정보 조회
PATCH /api/v1/mypage/account     // 계정 수정
PATCH /api/v1/mypage/account/password  // 비밀번호 변경

// 스터디 및 팀 엔드포인트도 존재
```

**수정**:
1. 각 마이페이지 컴포넌트에 API 클라이언트 import
2. useState 초기 데이터를 useEffect 내 API 호출로 교체
3. 저장 핸들러(alert())를 실제 API POST/PATCH 호출로 교체
4. 로딩 상태 및 오류 처리 추가
5. 성공/오류 토스트 알림 추가

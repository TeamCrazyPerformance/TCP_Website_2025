## 일반적인 문제 해결

### 백엔드가 시작되지 않음

**문제**: `Error: Cannot connect to database`

**해결책**:
1. 데이터베이스가 실행 중인지 확인: `docker ps`
2. `envs/db_dev.env` 자격 증명이 일치하는지 확인
3. `DB_HOST`가 올바른지 확인 (로컬은 `localhost`, Docker는 `db`)

---

### 프론트엔드가 API에 접근할 수 없음

**문제**: `API request failed: 404`

**해결책**:
1. API가 올바른 포트(3000)에서 실행 중인지 확인
2. `web/package.json`의 `proxy`가 API URL과 일치하는지 확인
3. `api/src/main.ts`의 CORS 설정 확인

---

### 인증이 작동하지 않음

**문제**: `401 Unauthorized`

**해결책**:
1. 환경에 JWT_SECRET이 설정되어 있는지 확인
2. 쿠키가 전송되고 있는지 확인 (브라우저 DevTools → Application → Cookies)
3. 토큰 만료 확인
4. 다시 로그인 시도

---

### Docker 컨테이너가 계속 충돌함

**문제**: 컨테이너가 계속 재시작됨

**해결책**:
1. 로그 확인: `docker logs <container-name>`
2. 환경 변수가 설정되어 있는지 확인
3. 헬스체크가 응답하는지 확인
4. 의존성이 먼저 시작되었는지 확인 (예: API 전에 DB)

## 향후 개선 사항

### 잠재적 기능

1. **실시간 기능**:
   - 실시간 알림을 위한 WebSocket 지원
   - 실시간 스터디 진행 상황 업데이트
   - 스터디 그룹을 위한 라이브 채팅

2. **분석**:
   - 회원 활동 추적
   - 스터디 완료율
   - 모집 전환 지표

3. **연동**:
   - 리포지토리 확인을 위한 GitHub API
   - 알고리즘 통계를 위한 백준 API
   - 중요 이벤트에 대한 이메일 알림

4. **모바일 지원**:
   - Progressive Web App (PWA)
   - 네이티브 모바일 앱 (React Native)

---

### 기술 부채

1. **프론트엔드 상태 관리**: 복잡한 상태를 위해 Redux 또는 Zustand 추가 고려
2. **API 버전 관리**: `/api/v1/` 버전 관리 구현
3. **속도 제한**: 남용 방지를 위한 속도 제한 추가
4. **캐싱**: 자주 접근하는 데이터에 대한 Redis 캐싱 구현
5. **파일 저장소**: 업로드를 클라우드 스토리지로 이전 (S3, CloudFlare R2)
6. **이메일 서비스**: 이메일 알림을 위한 SendGrid 또는 유사 서비스 통합

## 학습 경로

### 1단계: 환경 설정
- 개발 워크플로우 섹션을 따라 **로컬 개발 환경 설정**
- 애플리케이션을 실행하고 각 기능 테스트

### 2단계: 코드베이스 탐색
다음부터 **코드베이스 탐색** 시작:
- 프론트엔드: `web/src/App.js` → 라우팅
- 백엔드: `api/src/app.module.ts` → 모듈 임포트
- 데이터베이스: `api/src/members/entities/user.entity.ts` → 데이터 모델

### 3단계: 문서 읽기
- 작업할 영역에 대해 **모듈별 API 문서** 읽기
- 관련 아키텍처 섹션 검토 (프론트엔드 또는 백엔드)

### 4단계: 코드 작성
- 작은 변경사항으로 시작
- 추가하는 새 기능에 대해 **테스트 작성**
- 코드 스타일 가이드 따르기
